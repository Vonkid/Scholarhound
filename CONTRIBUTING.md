# Contributing

Thank you for helping improve ScholarHound.

1. Open an issue before substantial architectural changes.
2. Create a focused branch and include tests for behavioral changes.
3. Run `python -m pytest -q` before submitting a pull request.
4. Keep LLM-produced judgments explicit, provenance-tagged, bounded, and revisable.
5. Do not commit credentials, private corpora, copyrighted papers, participant data, local absolute paths, or production server configuration.

By contributing, you agree that your contribution is licensed under the MIT License.
